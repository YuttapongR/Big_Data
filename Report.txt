อยากได้โปรเจกเกี่ยวกับวิชา big data โดยให้ใช้ข้อมูลจาก kaggle โดยต้องใหญ่กว่า 10 Gb
โครงสร้างโปรเจกคือ
1.ต้องมีการแปลง/ทำความสะอาดข้อมูล
2.เขียนโดยใช้ pysprak
3.ควบคุมแต่ละขั้นตอนโดยใช้ airflow local
4.แสดงผลออกมาเป็นรูปแบบ dashboard
5.แยกโฟล์เดอร์ให้ชัดเจนว่าหน้านี้ทำหน้าที่อะไร
6.ต้องแสดงกราฟในหน้าเว็ปได้
7.เว็ปต้องสเถียรและอัพเดตค่าตลอดตามข้อมูลที่เปลี่ยนแปลง

หลังจากเจนเสร็จพบปัญหาairflowไม่เชื่อมต่อกับdashboradและไฟล์ Docker-compose.yml  อยู่ในโฟล์เดอร์ทำให้ใช้คำสั่งDockerไม่ได้จึงพรอมไปว่า

1.ช่วยแก้ไขให้airflowกับหน้าdashboradเชื่อมต่อกันได้และหาข้อบกพร่องในโปรเจกนี้และย้ายไฟล์ Docker-compose.yml และ.envออกจากโฟล์เดอร์

หลังจากแก้เสร็จพบว่าแชททำการสร้างvolumedockerให้ใหม่จึงพรอมไปว่า

1.สามารถ compose down อันเก่าและค่อย build อันใหม่ได้ไหม

แต่ละตาราง

การเชื่อมต่อในโปรเจกต์นี้แบ่งออกเป็น 2 ส่วนหลักๆ คือการเชื่อมข้อมูล "ภายใน Dataset" และการเชื่อม "ระบบหลังบ้านกับหน้าจอ" ครับ มีรายละเอียดดังนี้:

1. การเชื่อมข้อมูลใน Dataset (ระหว่างไฟล์ CSV)
เราใช้คอลัมน์ appid เป็น "กุญแจหลัก" ในการเชื่อมครับ:

Reviews + Applications: เราเอา appid จากไฟล์รีวิว ไปค้นหาในไฟล์แอปพลิเคชัน เพื่อให้รู้ว่ารีวิวนั้นคือ "ชื่อเกมอะไร" (เช่น appid: 10 = Counter-Strike)
ผลลัพธ์: ทำให้หน้า Dashboard แสดงผลเป็น "ชื่อเกม" แทนที่จะเป็นแค่ตัวเลข ID ครับ
2. การเชื่อมข้อมูลเข้าสู่หน้า Dashboard (แต่ละช่อง)
Dashboard จะดึงข้อมูลจากไฟล์ที่ Airflow รันเสร็จแล้วในโฟลเดอร์ data/processed/ มาแสดงในแต่ละส่วนดังนี้ครับ:

ส่วนบน Dashboard	ไฟล์ที่ใช้เชื่อม	ข้อมูลที่ดึงมาแสดง
KPI Cards (4 ช่องบน)	summary.json จำนวนรีวิวทั้งหมด, จำนวนเกมทั้งหมด, และจำนวนผู้เล่นทั้งหมดใน Dataset
Line Chart (กราฟเส้น)	daily_steam_reviews.parquet	วันที่รีวิว (review_date) เชื่อมกับจำนวนรีวิวบวก/ลบ เพื่อดูแนวโน้มรายวัน
Bar Chart (กราฟแท่ง)	top_games.parquet	ชื่อเกม (name) เชื่อมกับจำนวนรีวิวทั้งหมดทั่วโลก (recommendations_total)
Pipeline Status (แถบสถานะ)	.pipeline_status.json	สถานะการรันล่าสุด (status) และเวลาที่รันเสร็จ (last_run)
สรุปภาพรวม:
กุญแจเชื่อมข้อมูล: คือ appid
กุญแจเชื่อมระบบ: คือ โฟลเดอร์ data/processed/ (Airflow มีหน้าที่ "เขียน" ข้อมูลลงไป ส่วน Dashboard มีหน้าที่ "อ่าน" ขึ้นมาโชว์ครับ)

คำอธิบายแต่ละขั้นตอน (Step-by-Step Flow)

1. 📁 Data Source (ข้อมูลต้นทาง)

ผู้ใช้นำไฟล์ reviews.csv และ applications.csv จาก Kaggle ไปวางในโฟลเดอร์ data/raw/

2. ⚙️ Data Pipeline (การทำความสะอาดและสรุปผลด้วย PySpark)

Airflow จะสั่งรันสคริปต์ PySpark ตามลำดับดังนี้:

  1. clean_data.py:
    • อ่าน reviews.csv (จำกัด 500,000 แถวแรก) และทิ้งแถวที่โครงสร้างพัง (DROPMALFORMED)
    • สร้างไฟล์สรุปการทำความสะอาด data_quality_log.json ไว้ดูว่าข้อมูลเสียไปกี่แถว
    • บันทึกไฟล์ที่สะอาดแล้วเป็น Parquet (อ่านไว ใช้พื้นที่น้อย)
  2. aggregate_data.py:
    • นำไฟล์ที่สะอาดแล้วมา Group By เพื่อคำนวณจำนวนรีวิวเฉลี่ยรายวัน และสรุปผลภาพรวม
    • Games Analytics: คำนวณรายได้โดยประมาณ, อัตราความชอบ, และปีที่วางจำหน่ายแยกตามรายเกม
    • NLP Keyword Extraction: ใช้เทคนิคการแยกคำ (Tokenize) และกรอง Stop Words เพื่อหา "คำยอดฮิต" ในคอมเม้น
    • บันทึกผลลัพธ์ลงเป็นไฟล์ Parquet แยกตามประเภทการวิเคราะห์
    • สร้างไฟล์ summary.json และ .pipeline_status.json บอกว่ารันเสร็จสมบูรณ์

3. 🚀 Backend (ตัวกลางส่งข้อมูล)

FastAPI ทำหน้าที่เป็นประตูส่งข้อมูลจากไฟล์ Parquet/JSON เข้าสู่หน้าเว็บ:

  • /api/dashboard-data: ดึงข้อมูลแนวโน้มรายวัน (โดยระบบจะส่งข้อมูล 90 วันล่าสุดกลับมาให้อัตโนมัติเพื่อแสดงแนวโน้มกิจกรรม)
  • /api/games-analytics: ส่งข้อมูลเกม 5,000 อันดับแรกพร้อมข้อมูลเชิงลึก (ราคา, รายได้, คะแนน) สำหรับวาดกราฟและตาราง
  • /api/common-words: ส่งข้อมูลคำนิยม (Keywords) ที่ได้จากกระบวนการ NLP
  • /api/data-quality: ส่งค่า Log คุณภาพข้อมูลให้หน้าเว็บแสดงผล
  • /api/pipeline-status: ส่งสถานะการทำงานล่าสุดของ Spark Pipeline

4. 💻 Frontend (หน้าจอผู้ใช้)

  • app.js จะดึงข้อมูล (Fetch) จาก Backend ทั้ง 4 API มารวมกัน
  • ระบบ Slicers อัตโนมัติ: Backend จะส่งข้อมูลล่าสุดให้ทันที และใช้ Slicers (แนวเกม/ปี/ราคา) ในการกรองข้อมูลแบบ Real-time บนหน้าเว็บ
  • นำข้อมูลไปแสดงผลบนกราฟต่างๆ:
    • Reviews Trend: กราฟเส้นแสดงแนวโน้มรีวิวรายวัน
    • Market Analysis: กราฟแสดงส่วนแบ่งตลาดแยกตามแนวเกมและราคา
    • Technical Insights: กราฟสัดส่วนเกมฟรี vs เกมจ่ายเงิน (แทนที่ OS Support)
    • Trending Keywords: กราฟแท่งแสดงคำยอดนิยมจากรีวิว (NLP)
    • Game Explorer: ตารางค้นหาข้อมูลเกมที่ละเอียดและแม่นยำ

✨ อัปเดตระบบล่าสุด (Latest System Updates)

การปรับปรุงฟีเจอร์และการเพิ่มประสิทธิภาพของระบบที่ดำเนินการล่าสุด:

  • การปรับปรุงประสิทธิภาพ Pipeline (OS Data Removal): ถอดการวิเคราะห์ OS Support (Windows/Mac/Linux) ออกเพื่อลดขนาดข้อมูลและเพิ่มความเร็วในการประมวลผล เนื่องจากข้อมูลต้นทางมีความคลาดเคลื่อนสูง
  • Technical Metric ใหม่ (Free vs Paid): เปลี่ยนจากการดู OS มาเป็นการวิเคราะห์สัดส่วน "เกมฟรี vs เกมจ่ายเงิน" ในแท็บเทคนิค เพื่อให้นักพัฒนาเข้าใจโครงสร้างราคาในตลาดได้ชัดเจนยิ่งขึ้น
  • ความแม่นยำของข้อมูลใน Game Explorer: ปรับปรุงชื่อคอลัมน์เป็น "รีวิวที่วิเคราะห์" (Dataset) และ "ยอดแนะนำ (Steam)" (Official) พร้อมบังคับประเภทข้อมูลเป็น Integer ใน PySpark เพื่อให้การเรียงลำดับ (Sorting) ถูกต้องแม่นยำ 100%
  • Consistency Fix (Sorting & UI): แก้ไข Bug การเรียงลำดับเริ่มต้นให้ตรงกันระหว่าง Spark Job และไอคอนบนหน้าเว็บ (Default: รีวิวที่วิเคราะห์ DESC)
  • UI Simplification: ลบตัวกรองวันที่ (Start/End Date) ออกเพื่อลดความซับซ้อนของส่วนควบคุม และใช้ระบบดึงข้อมูล 90 วันล่าสุดโดยอัตโนมัติแทน
  • Codebase Housekeeping: ลบสคริปต์ Hotfix และไฟล์ชั่วคราวที่ไม่จำเป็นออก เพื่อรักษาความสะอาดและความเป็นระเบียบของโครงสร้างโปรเจกต์

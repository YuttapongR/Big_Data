คำอธิบายแต่ละขั้นตอน (Step-by-Step Flow)

1. 📁 Data Source (ข้อมูลต้นทาง)

ผู้ใช้นำไฟล์ reviews.csv และ applications.csv ไปวางในโฟลเดอร์ data/raw/

2. ⚙️ Data Pipeline (การทำความสะอาดและสรุปผลด้วย PySpark)

Airflow หรือการรันด้วยมือจะสั่งรันสคริปต์ PySpark ตามลำดับดังนี้:

*หมายเหตุ: สคริปต์รองรับสถาปัตยกรรม Relative Paths ทำให้รันได้ทั้งบน Local (Windows) และ Docker โดยไม่ต้องแก้โค้ด

  1. clean_data.py:
    • อ่าน reviews.csv (จำกัด 500,000 แถวแรก) และทิ้งแถวที่พัง (DROPMALFORMED)
    • สร้างไฟล์สรุปการทำความสะอาด data_quality_log.json เพื่อตรวจสอบคุณภาพข้อมูล
    • บันทึกไฟล์ที่สะอาดแล้วเป็น Parquet ไว้ใน data/processed/
  2. aggregate_data.py:
    • นำไฟล์ที่สะอาดแล้วมาคำนวณสถิติรายวัน และสรุปผลภาพรวม
    • Games Analytics: คำนวณรายได้โดยประมาณ, อัตราความชอบ, จำนวนภาษา และปีที่วางจำหน่าย (จำกัด 5,000 อันดับแรกเพื่อประสิทธิภาพ)
    • NLP Keyword Extraction: กรอง Stop Words เพื่อหาคำยอดนิยมจากรีวิว
    • บันทึกผลลัพธ์เป็นไฟล์ Parquet สำหรับ API Backend

3. 🚀 Backend (ตัวกลางส่งข้อมูล)

FastAPI ทำหน้าที่อ่านไฟล์ Parquet จากโฟลเดอร์ data/processed/ และส่งข้อมูลผ่าน API:

  • /api/dashboard-data: ดึงข้อมูลสรุปรายวันและ KPI หลัก
  • /api/games-analytics: ส่งข้อมูลเกมพร้อมฟิลด์ที่ผ่านการคำนวณ (ราคา, รายได้, คะแนน, ปี)
  • /api/common-words: ส่งข้อมูลคำนิยมสำหรับ Word Cloud
  • /api/data-quality: ส่งสถิติการทำความสะอาดข้อมูล (Cleaned vs Dropped)
  • /api/pipeline-status: ส่งสถานะการทำงานล่าสุดของ Spark Pipeline

4. 💻 Frontend (หน้าจอผู้ใช้)

  • Consolidated UI: ยุบรวมข้อมูลเชิงพาณิชย์และแนวโน้มเข้าด้วยกันเพื่อให้อ่านง่ายในหน้าเดียว
  • นำข้อมูลไปแสดงผลบนกราฟต่างๆ:
    • Release Trend: กราฟเส้นแสดง "จำนวนเกมที่วางจำหน่ายจริง" ตามช่วงเวลา
    • Market Analysis: กราฟแท่งแสดงส่วนแบ่งตลาดแยกตามแนวเกม
    • Commercial Insights: กราฟสัดส่วนเกมฟรี vs เกมจ่ายเงิน (ย้ายจากแท็บเทคนิคมาไว้หน้าแรก)
    • Price vs Score: กราฟจุด (Scatter) วิเคราะห์ความคุ้มค่าของราคาต่อคะแนนรีวิว
    • Game Explorer: ตารางสำรวจข้อมูลเกมแบบ Interactive

✨ อัปเดตระบบล่าสุด (Latest System Updates)

  • Hybrid Architecture Support: ปรับปรุงระบบจัดการพาธให้เป็นแบบ Relative ทำให้รัน Pipeline ได้ทั้งบน Local (venv) และ Docker/Airflow โดยไม่ต้องสลับโค้ดไปมา
  • UI Consolidation & Expansion: ลบแท็บ "ข้อมูลทางเทคนิค" และย้ายกราฟ "สัดส่วนเกมฟรี/จ่ายเงิน" มาไว้ที่หน้าแรก (Market & Trend) พร้อมขยายขนาดกล่องให้ใหญ่ขึ้น
  • Release Trend Correction: แก้ไขตรรกะกราฟแนวโน้มให้แสดง "จำนวนเกมที่วางจำหน่ายตามช่วงเวลา" จริงๆ (อ้างอิงจาก Release Year)
  • Performance Optimization: คงลิมิตการส่งข้อมูลเกมไว้ที่ 5,000 อันดับสูงสุด เพื่อความลื่นไหลของหน้า Dashboard
  • System Cleanup: ลบไฟล์ Parquet ที่ล้าสมัยและโค้ดที่ไม่ได้ใช้งาน (Dead Code) ออกจากโปรเจกต์
